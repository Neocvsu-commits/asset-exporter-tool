# 资产导出助手 v2.5.1

把 Blender 资产按团队规范一键导出——FBX、GLB、贴图、报告、Blend 备份，选好目录点一下就行。

## 解决的问题

模型做完到交付，导出这步经常出状况：参数忘了调、贴图没带上、审查报告漏了、多格式导出顺序不对导致材质丢失。手动一项项检查费时，偶尔还会少导出东西。

这个插件把导出流程标准化了。选好模型 → 勾上要导出的内容 → 点导出，FBX、GLB、贴图、基础信息报告、审查报告、Blend 备份一起出。

## 核心功能

### 📦 导出模式

**合并导出** — 多个选中对象导出到同一个 FBX/GLB 文件，文件内保持多个独立物体，各自保留原名（不会 Join 成一个网格）。

**逐个导出** — 每个选中对象独立导出同名文件，重名自动加后缀。

### 📦 导出结构

**打包导出** — 每个资产一个外层文件夹，贴图、报告、模型都收在里面。多资产时结构清晰。

**直接导出** — 不建外层文件夹，文件直接写入你选的目录。贴图目录写为 `<资产名>_Texture`，防止多资产时互相覆盖。

### 📐 正前方向

面板中设置模型正前方向，插件会创建参考箭头并写入对象属性。未设置时会高亮提醒，报告里也会记录这个方向信息。

### 📋 基础信息报告

勾选后自动输出两份文件：`<资产名>_BasicInformation.csv` 和 `<资产名>_BasicInformation.json`。内容包含尺寸、三角面数（`loop_triangles`，多物体汇总）、材质与贴图统计、Transform 检查、骨骼/动画状态、正前方向、全局旋转四元数等。

CSV 中文展示，JSON 英文键。两份文件由同一数据源生成，不会出现数据不一致。

### 🔗 资产审查联动

如果装了重构版 `资产审查助手`，导出时会自动读取审查结果并输出审查 CSV/JSON。支持别名、大小写/空白兼容，物体改名时按数量对齐自动匹配（不再限于单物体场景）。审查结果不可用时仅警告不阻断，不影响主模型导出。

### 💾 Blend 备份 + 贴图提取

- **备份 Blend 工程**：输出仅含导出资产和依赖数据的 `.blend`
- **导出提取贴图**：提取材质节点中的 `TEX_IMAGE` 到 `Texture` 目录，支持以下特性：
  - **内存优先写出**：缩放/格式转换后的贴图从当前像素写出，而非复制磁盘旧文件
  - **扩展名正确替换**：修复旧版双后缀 bug（`T_xx_D.tga.targa`），正确擦除旧扩展名
  - **多级兜底**：`save_render` → packed 字节直接写出 → `shutil.copy2` 盘文件

### 🛡️ GLB 导出前的空图像修复

PBR 贴图助手改格式/重命名后可能残留 `.001` 空壳 Image 数据块（无像素数据），glTF 导出器的 `make_temp_image_copy` → `copy()` → `update()` 流程遇到此类数据会崩溃。

导出助手在 GLB 导出前自动执行三层修复：
1. 按物体名前缀匹配有效同名贴图，替换空壳引用
2. 无法匹配时尝试 `pack()` 从源路径重新加载
3. 以上均失败时移除空图像节点

确保 GLB 导出不因贴图数据块异常而中断。

## FBX / GLB 参数说明

### 设计原则

`更多 FBX 参数` 和 `更多 GLB 参数` 打开的是 Blender 原生导出器面板——不是自定义字段，层级和逻辑跟原生保持一致。在这里调好的参数会被统一导出流程读取。

### 打开参数弹窗时的预填值

点击"更多参数"时会先填入推荐值，你可以手动改：

**FBX：**
- 仅选中物体、物体类型默认网格
- 三角化
- 贴图路径剥离，不嵌入贴图
- 应用修改器、切线空间、单位/空间变换按当前策略预填

**GLB：**
- 导出格式 GLB
- 仅选中物体
- 网格和材质相关项按当前策略预填

### 统一导出时的强制项

为保证流程稳定，以下项在导出时强制锁定：

- **FBX**：`use_selection=True`、`object_types={"MESH"}`、`path_mode="STRIP"`、`embed_textures=False`
- **GLB**：`export_format="GLB"`、`use_selection=True`

同时导出 FBX 和 GLB 时，插件会先出 GLB 再出 FBX。FBX 的纯模型处理（剥离贴图节点）放在 GLB/Blend 之后执行，避免影响其他格式的材质导出。

## 快速上手

1. 在视图中选中要导出的 MESH 物体
2. 选择导出模式：`合并导出` 或 `逐个导出`
3. 选择导出结构：`打包导出` 或 `直接导出`
4. 勾选需要导出的内容
5. 需要调整 FBX/GLB 参数时，点 `更多 FBX 参数` / `更多 GLB 参数`
6. 点 `选择目录并导出`，选导出根目录
7. 导出后点 `打开上次导出目录` 快速定位

## FAQ

**Q1：审查 CSV/JSON 为什么是灰色的？**
通常是以下原因之一：没装资产审查助手、还没执行检查、当前选中对象没被这次审查结果覆盖。

**Q2：点"更多参数"会直接导出吗？**
不会。更多参数只是配置面板，真正导出只在点 `选择目录并导出` 时触发。

**Q3：能快速回到上次导出目录吗？**
点 `打开上次导出目录`。目录不存在时按钮灰置。

**Q4：为什么导出后面数和场景里看到的不一样？**
导出流程会在副本对象上应用变换并三角化，以导出副本的结果为准。

**Q5：打包导出和直接导出怎么选？**
每个资产需要一个完整文件夹就用 `打包导出`；只要 FBX/GLB/贴图文件、不需要外层文件夹就用 `直接导出`。

**Q6：直接导出会覆盖同名文件吗？**
不会。检测到同名冲突时自动追加 `_001`、`_002` 后缀避让。

## 版本迭代规则

遵循语义化版本（SemVer）`MAJOR.MINOR.PATCH`，唯一版本源在 `__init__.py` 的 `bl_info["version"]`：

| 改动类型 | 改哪一位 | 举例 |
|---|---|---|
| 修 bug、调默认值、改文案 | PATCH | `2.3.8 → 2.3.9` |
| 加新功能，向后兼容 | MINOR | `2.3.9 → 2.4.0` |
| 大改架构、破坏旧版兼容 | MAJOR | `2.4.0 → 3.0.0` |

- MINOR 升位时 PATCH 归零，MAJOR 升位时 MINOR 和 PATCH 都归零
- 发布 GitHub Release 时 tag 命名加 `v` 前缀：`v2.3.9`
