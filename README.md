# 资产导出助手（团队内部使用文档）

## 1. 插件定位
资产导出助手用于把 Blender 资产按团队规范一键导出为交付包，核心目标是：

- 模型导出流程统一（FBX / GLB）。
- 导出参数可调且与 Blender 当前版本导出器保持一致。
- 基础信息、审查报告、贴图提取、Blend 备份一并处理。
- 在“审查不可用”时不阻断主导出流程。

## 2. 基本信息

- 插件版本：`2.3.1`（正式版）
- Blender 支持：`3.6+`
- 入口：`3D 视图 > N 面板 > Asset Export`

## 3. 标准使用流程（推荐）

1. 在视图中选中要导出的 `MESH` 物体。
2. 选择导出模式：`合并导出` 或 `逐个导出`。
3. 选择导出结构：`打包导出` 或 `直接导出`。
4. 根据需要勾选导出项（FBX / GLB / 基础报告 / 审查报告 / Blend / 贴图）。
5. 如需调整 FBX/GLB 参数，点击：
   - `更多 FBX 参数`
   - `更多 GLB 参数`
6. 点击 `选择目录并导出`，选择导出根目录并执行。
7. 导出后可点 `打开上次导出目录` 快速定位结果。

## 3.1 本次版本更新（2.3.1）

- FBX 导出策略升级为“纯模型优先”：
  - 导出时强制剥离贴图路径（`path_mode="STRIP"`）
  - 禁用贴图嵌入（`embed_textures=False`）
  - 在导出副本上移除材质中的贴图节点引用，避免回导 FBX 出现粉色材质依赖
- 适配场景：ORM/ARM 等分离贴图工作流，贴图由后续规则进行重链接而非依赖 FBX 内记录路径。

## 4. 导出参数（FBX / GLB）

### 4.1 设计原则
- `更多 FBX 参数` 和 `更多 GLB 参数` 打开的不是自定义字段集合，而是基于 Blender 当前版本导出器参数绘制。
- 层级与逻辑尽量忠实原生导出器（包括分组结构、启用关系等），降低学习成本。
- 参数在这里调整后，会被统一导出流程读取并生效。

### 4.2 打开参数弹窗时的推荐默认值
为保证第一次使用更贴近团队规范，点击“更多参数”时会先预填关键项：

- FBX：
  - 仅选中物体（`use_selection=True`）
  - 物体类型默认网格（`object_types={"MESH"}`）
  - 三角化（`use_triangles=True`）
  - 贴图路径模式默认剥离（`path_mode="STRIP"`）
  - 默认不嵌入贴图（`embed_textures=False`）
  - 应用修改器、切线空间、单位/空间变换等按当前导出策略预填
- GLB：
  - 导出格式默认 `GLB`
  - 仅选中物体（`use_selection=True`）
  - 网格与材质相关常用项按当前导出策略预填

> 说明：这些是“预填默认值”，可在弹窗中手动改。最终以你在参数弹窗中的设置为准。

### 4.3 统一导出阶段的强制项
为保证流程稳定，统一导出时会固定以下规则：

- FBX：强制 `use_selection=True`、`object_types={"MESH"}`、`path_mode="STRIP"`，并禁用 `embed_textures`
- GLB：强制 `export_format="GLB"` 且 `use_selection=True`

## 5. 主要功能说明

### 5.1 导出模式
- `合并导出`：选中对象复制后合并，作为一个资产包导出。
- `逐个导出`：每个选中对象独立导出为单独资产包。

### 5.2 模型正前方向
- 面板中可设置正前方向，插件会创建/更新参考箭头并写入对象属性。
- 未设置时会高亮提示。
- 报告会写入该方向信息。

### 5.3 导出结构
- `打包导出`：每个资产创建独立外层文件夹（团队原有默认行为）。
- `直接导出`：不创建外层文件夹，直接把文件写入你选择的目录。
- 直接导出模式下，贴图目录会写为 `<资产名>_Texture`，用于避免多资产导出时贴图互相覆盖。

### 5.4 基础信息报告
若勾选对应选项，会输出：

- `<资产名>_BasicInformation.csv`
- `<资产名>_BasicInformation.json`

内容包含：尺寸、三角面数、材质与贴图统计、Transform 检查、骨骼/动画状态、正前方向、全局旋转四元数等。

### 5.5 资产审查联动
支持与重构版 `资产审查助手` 联动：

- 优先读取新版审查结果（`results_json`）。
- 未命中新版时可兼容旧版审查数据源。
- `导出资产审查 CSV/JSON` 仅在“当前选中对象已完成审查”时可用。
- 审查不可用时会跳过审查文件，不影响主模型导出。

### 5.6 Blend 与贴图
- `备份 Blend 工程`：输出仅含导出资产及依赖数据的 `.blend`。
- `导出提取贴图`：提取材质节点中的 `TEX_IMAGE` 到 `Texture` 目录。

## 6. 默认开关（当前）

- 导出 FBX：开
- 导出 GLB：开
- 导出基础信息 CSV：开
- 导出基础信息 JSON：开
- 导出资产审查 CSV：开（审查不可用时灰色）
- 导出资产审查 JSON：开（审查不可用时灰色）
- 备份 Blend 工程：开
- 导出提取贴图：开

├─ Texture/

## 8. 常见问题（FAQ）

### Q1：为什么审查 CSV/JSON 是灰色？
A：通常是以下原因之一：
- 未安装或未启用资产审查助手；
- 尚未执行检查；
- 当前选中对象没有被这次审查结果完全覆盖。

### Q2：更多参数会不会直接导出文件？
A：不会。更多参数用于配置，真正导出只在 `选择目录并导出` 执行。

### Q3：导出目录能快速回到上一次吗？
A：可以，点 `打开上次导出目录`。目录不存在时按钮会灰色不可用。

### Q4：为什么有些模型导出后面数与场景里不一致？
A：导出流程会在副本对象上应用变换，并尝试三角化，结果以导出副本为准。

### Q5：打包导出和直接导出怎么选？
A：需要“每个资产一个完整包”就用 `打包导出`；只想快速拿 `FBX/GLB/贴图` 文件、不需要外层文件夹就用 `直接导出`。

### Q6：直接导出会覆盖同名文件吗？
A：不会。若检测到同名冲突，导出会自动追加 `_001`、`_002` 等后缀避让。
